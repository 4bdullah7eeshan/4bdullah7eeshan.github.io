## When to use?

If an entity/note is supposed to be in only one fixed place within a hierarchy and nowhere else, we can use them. 

## When not-to use?

Some notes want to be free and not rigid in a hierarchy.  In such cases we must not use them.

Another way to decide is if one note can be put in multiple places under a parent note, we shouldn't use hierarchy
