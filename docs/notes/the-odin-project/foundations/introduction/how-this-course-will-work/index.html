How This Course Will Work

Abdullah Zeeshan

Introduction

The Odin Project is an open-source community dedicated to providing the best information sources to take you from zero to a full-stack developer.

The lesson begins by telling us the following two things:

  1. What is The Odin Project?
  2. What they do?

What is The Odin Project?

With regards to this, they say that:

The Odin Project is an open-source community…

Let us break this down.

community
a group of people.
open-source community
a community(a group of people) that is involved in the development of open source software.

a software whose code us made publicly available.

In conclusion, The Odin Project is a group of people that are involved in the development of open-source software.

What open-source software does The Odin Project develop?

The above question arises naturally.

I think it is this curriculum itself.

What they do?

The next bit is around what they do. They are

…dedicated to providing the best information sources to take you from zero to a full-stack developer.

The journey from a zero to a Full Stack Developer

At the moment of starting this course, most of us are a zero at web development. From this zero, we would like to become a Full Stack Developer.

Many ways

In order to make this transition happen, there can be many ways:

  • Way 1
  • Way 2

One way: Learn from the best sources

One of the ways is learning from the best sources. Sources that already exist on the internet and else where.

However, to learn from the best sources, one must first get them. This involves a lot of work which I am not going to go into right now.

Folks at The Odin Project are dedicated to getting and providing these to us.

What we will do in the Foundations course?

In this unit, we’ll learn about how the web works and start thinking about the basics of computer and web programming.

Next, they tell us what we will do in this unit.

Unit means course.

Here, when they say this unit, they are referring to the complete Foundations course.

The web has a working of its own.

In the Foundations course, we will do the following two things overally:

  1. Learn about how the web works
  2. Start thinking about the basics of computer and web programming

Basics does not mean simple.

basic
forming an essential foundation or starting point; fundamental.

The Foundations course covers baseline knowledge of web development

Each of the following sections and lessons covers essential baseline knowledge.

baseline knowledge
the minimum level of knowledge/understanding/skill that one must possess in a given setting.

How is this useful?

Even if you have no intention of becoming a web developer, this material should help you gain a useful understanding of the moving parts involved in creating and serving content on the web.

There are various moving parts involved in creating and serving content on the web.

Even if we have no intention of becoming a web developer, the stuff covered in the Foundations course should help us gain a useful understanding of these moving parts.

Learnings goals and plan

We will start by getting familiar with the internet and your own computer. Next, we’ll set up a development environment and learn about Git and GitHub. Then, we’ll go over the basics of front-end technologies like HTML, CSS, and JavaScript before stepping into the back end with a brief foray that covers the basic concepts of back-end technologies.

Below is an orderly list of things we will do in the Foundations course:

  • Get familiar with the internet
  • Get familiar with our own computer
  • Set up a development environment
  • Learn about Git
  • Learn about GitHub
  • Go over the basics of the following front-end technologies:
    1. HTML
    2. CSS
    3. JS
  • Step into the back end with a brief raid that covers the basic concepts of back-end technologies.

Learning outcomes of the Foundations course

By the end of this unit, you should not only understand how the web works but also be able to identify and differentiate between all the technologies that you will use to build your own web applications. You will be able to build a basic webpage, style it, and add minor elements of interactivity while working comfortably from the command line.

First let us know the following three points:

  1. The web has a working of its own (as mentioned earlier).
  2. To build web applications various technologies are used.
  3. interactivity
    the ability of a computer to respond to a user’s input

Now, let us look at the learning outcomes.

By the end of this unit(Foundations course), we will:

  • Understand how the web works.
  • Be able to identify and differentiate between all the technologies that are used to build web applications.
  • Be able to build a basic webpage, style it, and add minor elements of interactivity while working comfortably from the command line.

How it works

By it here, they mean the The Odin Project’s curriculum.

This section is about “How The Odin Project’s curriculum works”.

This curriculum works by aggregating the best content from across the internet to teach a specific topic. In each lesson, we’ll introduce the topic and try to provide some useful context before pointing you to external resources made by others.

Consider a specific topic, let us say How the web works?.

There exists various content on the internet around that topic. They collect and gather the best of these.

Each topic is taught in a lesson.

In a lesson, they:

  1. Introduce the topic to us
  2. Try to provide some useful context
  3. Point us to the external resources made by others.

Questions, Exercises, and Projects

Most lessons will contain questions that you should be able to answer before moving on. Some lessons will include exercises for you to complete. In addition, we provide several projects throughout the curriculum to help you grow your understanding by actually building things.

The questions in a lesson are like a checklist to tick-off and act like a measuring tool if we have completed the lesson.

If we can comfortably answer the questions, we can safely move to the next lesson.

What not to think of The Odin Project or programming?

Try not to think of The Odin Project, or programming, as a class in school. It’s not material you learn all at once to take a test, and then pass or fail.

They are associating The Odin Project with programming. When we are doing The Odin Project, we are essentially doing programming.

They say that we should not think of The Odin Project, or programming, as a class in school.

What happens in a class in school?

In a typical class in school, we learn some material all at once to take a test, and then pass or fail.

Programming is not like this - learning some material all at once!

What to think of The Odin Project or programming?

You can think of it as a snowball. You, yourself, are a snowball. You’re rolling down a hill full of snow; the further you roll, the more snow will stick to you. Sure, snow will also fall off you, and you’ll forget things often, but that’s just part of the process. Don’t worry if you reach a project and feel like you haven’t retained or memorized anything. That’s natural and happens to everyone. The projects are not tests of what you have memorized so far. They are practice points where the information will come back to you as you start solving your problems one at a time, relying on Google and the Odin Community for help.

I like this idea.

Turns out it is called as Snowball effect.

Also, they are not emphasizing on memory at all!

Getting into a position where you are comfortable with the skills will mean that you will also be comfortable at expanding your skill-set. Once you have a solid base, it will be simpler to expand upon it. This is exactly what The Odin Project is about: developing these skills.

There are various parts of this course.

Some of these parts are the most difficult ones.

One of these is adopting the right mindset.

AZ: How do we adopt it?

An even bigger challenge is learning the core ideas of programming.

They say that programming is not just about coding. It is about the following four things:

  1. Coding
  2. Problem-solving
  3. Asking the right questions
  4. Doing enough research

After completing the course you will be in a position where you can simply jump straight into the documentation for anything you are trying to learn. Reading the documentation is the best way to learn any tooling. But it takes skill to master. Many people feel intimidated by the technical jargon, which is understandable. People who write documentations usually assume a background in programming. This is exactly why we give you plenty of material to read on your own.

The Odin Project makes us capable of learning a new language or a tool independently directly from the official documentation itself.

The skills you will gain from completing The Odin Project will be the foundation that you will be building upon for years and decades to come. If you come out of the course thinking that you need another course like this one to learn something like Python, then you either don’t believe in yourself or you haven’t taken away the important ideas that are covered in this course.

The skills you learn here are transferable to various contexts. Once you know the idea and that it is possible to do something, actually finding out how to do that thing becomes significantly easier and is simply a Google search away most of the times.

Imagine that we are in a new context. All that matters are the following things:

  1. What to do? (The idea)
  2. Is it possible to do it?
  3. Finding out how to do it.

The last step is simply a Google search away most of the time. But figuring out the first two is what we must master.

Translations

The maintainers are:

  1. Not able to

Why English?

The programming world is rooted in the English language

As you develop into a programmer, you will find that the world you are entering into is firmly rooted in the English language. This means that the syntax of your programming language, the documentation that teaches you how to use it, and the majority of the people in the community are all expecting to communicate with you in English.

To the non-English speakers

If you are a non-English speaker, or English is not a primary language for you, this fact is not meant to discourage you, but to prepare you.

If some of us are any of the following two categories:

  1. non-English speakers
  2. English is not our primary language,

In addition to this, you might also consider using a translation dictionary in your own language alongside our curriculum so you can readily reference it as you go.
